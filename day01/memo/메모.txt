기본 브랜치

HEAD -> master 의미
HEAD가 위치한 범위가 작업 시점

동일한 브랜치에서 작업하면 작업이 엉킬 수 있다.
브랜치를 나누고, 따로 작업하여 나중에 합칠 수 있다.

master는 기본 브랜치다.

작업하다보면 동일한 소스를 건드릴 수 있다.

원격 웹코드

원격 레포지토리 연결(원격 저장소 연결)

깃허브에 프로젝트 올리기...

다른 작업자가 원격 저장소에 연결하는 방법
1)git init
git remote add origin 원격 저장소 주소

원격 저장소 상태 -> 로컬 저장소 상태로 동기화
git pull origin 원격 저장소 브랜치명

무언가 수정사항이 발생하면 빨리 알려줘야 한다.
이전 화면을 보고있을수 있기 때문

git add는 작업대에 올린다 생각하면 된다.
스냅샷(수정사항 확인 등) 단계를 거쳐서 올라간다.
